# 自研链路能力边界报告

基于 8 条固定 prompt × 3 模型（Pretrain / SFT 全参 / LoRA）的实测数据。

评测时间：2026-07-24

---

## 一、Pretrain → SFT → LoRA 的能力差异

| 维度 | Pretrain | SFT 全参 | LoRA (0.60%) |
|------|------|------|------|
| 对话意识 | 无（纯续写/乱码） | 有（指令跟随） | 有 |
| 中文输出 | 符号乱码+英文混合 | 流畅中文 | 流畅中文 |
| 平均评分 | ~0.5 | **~3.0** | **~2.5** |
| 格式遵循 | N/A | 较好（编号、列表） | 概念解释类更强 |
| JSON 输出 | 不可能 | Parse%=0% | Parse%=0% |
| 长输出质量 | N/A | ~64-128 token 后崩坏 | 同左 |

## 二、全参 SFT vs LoRA 的分化

| | SFT 全参 (38M) | LoRA (229K, 0.60%) |
|------|------|------|
| val_loss | 2.45 | 2.58 (+5.3%) |
| 优势领域 | 指令遵循稳定、列表格式规范 | 概念解释结构化更强（机器学习 4 分） |
| 存储 | 434MB | **1MB (99.7% 节省)** |
| 共同弱点 | 128 token 后 repetition loop | 同左 |

单条 prompt 的典型差异：

| prompt | SFT 全参 | LoRA | 差异 |
|------|------|------|------|
| "解释机器学习" | 有结构、方向对，但重复 | 提到 ML/DL、分类、NLP/CV 等分支，结构更强 | LoRA > 全参 |
| "保持身体健康" | 9 条建议，饮食+睡眠+运动 | 7 条建议，内容接近 | 基本持平 |
| "写秋天的诗" | 前几句有意境，后半"又又又"重复 | 前几句也有意境，后半 emoji 崩溃 | 基本持平 |
| "四大发明" | 内容混乱，提到纸/火但全是瞎编 | 同左（38M 天花板） | 基本持平 |
| "Python 特点" | 结构有但内容不对 | 内容也偏 | 基本持平 |

## 四、能力边界总结

| 能做到 | 做不好 | 做不到 |
|------|------|------|
| 简单概念解释（AI 定义、机器学习原理） | 精确翻译 | 长文连贯叙事 |
| 格式化列表和编号回答 | 高质量诗歌 | JSON 结构化输出 |
| 多轮结构（编号 1-9） | 事实准确性 | 无 repetition 的长输出 |
| 基本对话格式（Human/Assistant） | 代码生成 | 多步推理 |

### Repetition Loop（重复崩溃）

SFT 和 LoRA 在多数 prompt 上 64-128 token 后进入重复模式：

| prompt | 崩溃表现 |
|------|------|
| 秋天的诗 | "又又又又" → emoji 循环 |
| 四大发明 | "****" → 空白字符 |
| Python | 重复输出格式模板 |
| 书籍推荐 | 内容重复 |

**这不是训练的问题，是模型容量的天花板。** 38M 参数 + 64 维 attention head 无法维持 128 token 以上的语义一致性。

## 五、核心结论

所有失败模式指向同一个根因——**模型容量不足**。

```
31.7M 参数
  ├── attention d_head=64  → 长程注意力退化
  ├── FFN dim=512          → 知识存储有限
  ├── layers=8             → 上下文抽象层次不够
  └── vocab=6400           → 编码精度有限，偶发乱码
```

这直接驱动了后续向 Qwen 迁移的决策。
